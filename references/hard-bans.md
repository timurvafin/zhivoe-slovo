# HARD BANS и детерминированные маркеры

Два уровня. **Уровень 0** — следы копирования из чат-бота: это не стиль, а технический мусор, один такой фрагмент = почти точно нередактированный вывод ИИ. **Уровень 1** — 20 конструкций, которые в неформальном/публицистическом русском запрещены всегда: не «исправляй», а удаляй и перестраивай фразу.

Исключения по жанрам — в `false-positives.md`. В научном, юридическом и художественном тексте часть запретов снимается.

---

## Уровень 0. Чат-бот-артефакты (regex)

Найден хоть один в тексте, претендующем на самостоятельное авторство, — удаляй маркер, ищи реальный источник, проверяй остальной текст. Это единственные **детерминированные** приметы ИИ.

| Маркер | Кто оставляет | Regex |
|---|---|---|
| `:contentReference[oaicite:N]{index=N}` | ChatGPT, внутренняя метка ссылки | `:contentReference\[oaicite:\d+\]\{index=\d+\}` |
| `oai_citation:N‡…`, `oaicite:N` | ChatGPT, варианты метки | `oai_?citation:\d+‡`, `oaicite:\d+` |
| `turn0search0`, `turn0fetch0` | ChatGPT, идентификатор веб-поиска | `turn\d+(search\|fetch)\d+` |
| `citeturn0file0`, `citeturn2search5` | ChatGPT, служебная метка цитаты из потока | `citeturn\d+[a-z]+\d+` |
| `?utm_source=chatgpt.com`, `?utm_source=openai` | ChatGPT, UTM-метка в ссылках | `[?&]utm_source=(chatgpt\.com\|openai)` |
| `【N†source】`, `【N:M†source】` | OpenAI Assistants, поиск по файлам | `【\d+(?::\d+)?†source】` |
| `[^N^]` (двойная каретка) | Microsoft Copilot / Bing, сноска | `\[\^\d+\^\]` |
| `](sandbox:/mnt/data/…)` | ChatGPT, сломанная ссылка на файл из контейнера | `\]\(sandbox:/mnt/data/` |
| `grok_card://…` | xAI Grok, карточка записи в X | `grok_card:\/\/` |
| `vertexaisearch.cloud.google.com/grounding-api-redirect/` | Google Gemini, ссылка веб-поиска | `vertexaisearch\.cloud\.google\.com/grounding-api-redirect` |
| `&#8217;` `&#8220;` `&#8221;` | старые модели, HTML-сущности вместо кавычек | `&#82(17\|20\|21);` |

**Старое поколение (2023-2024), ищи по тексту:** «На момент моего обучения…», «По состоянию на момент обновления моей базы…», «Я — языковая модель…», «As of my last knowledge update…». В свежем корпусе редко, в старых блогах и дипломах — массово.

**Границы.** Обычная сноска Markdown `[^1]` (одна каретка) — норма, не маркер. `[1]:`, `[2]:` — нормальная разметка ссылок, если в конце документа есть соответствующий список сносок. Слова «sandbox» и «/mnt/data» по отдельности в техническом тексте — норма; признак только внутри ссылки `](sandbox:/mnt/data/…)`.

**Подлог источника** (проверять при наличии ссылок): ссылка отдаёт 404; DOI ведёт на другую статью; ISBN не существует; автор умер до года публикации; ссылка на книгу без номеров страниц. Один подтверждённый подлог = текст машинный. Не можешь подтвердить факт — пометь как непроверенный, не выдавай за истину.

## Уровень 1. Двадцать запрещённых конструкций

Удаляй и перестраивай фразу. Это не стилистический выбор, а детектируемые маркеры — частотность некоторых в LLM-тексте в сотни раз выше человеческой.

| Конструкция | Почему | Чем заменить |
|---|---|---|
| «Не просто X, а Y» | фирменная формула GPT | прямо: «Y» без противопоставления |
| «Не только X, но и Y» | калька синтетического усиления | «X. И ещё Y» или просто перечислить |
| «В современном мире…», «В эпоху…» | пустое космическое открытие | начать с факта, числа, вопроса |
| «В условиях [прил.] [сущ.]…» | вариация того же пустого открытия | конкретика: что именно происходит |
| «Стоит отметить, что…» | калька с It's worth noting | убрать, сказать напрямую |
| «Важно понимать, что…» | калька с It's important to understand | убрать или «Тут вот что:» |
| «Данный / данная / данное» | канцелярский маркер | «этот / эта / это» |
| «Является» (>1 на 500 слов) | в русском связка в настоящем не нужна | перестроить, тире или «это» |
| «Играет важную/ключевую роль» | раздувание значимости | показать ПОЧЕМУ важно, через факт |
| «Можно с уверенностью сказать» | оговорка + раздувание | сказать без преамбулы |
| «Подводя итог», «Таким образом» (как вывод) | формульный вывод, клеится к чему угодно | убрать или начать вывод с действия |
| «И вот здесь начинается самое интересное» | калька-маркер ChatGPT | удалить, сразу писать интересное |
| «Погрузимся в…», «Давайте разберёмся» | артефакт чатбота, анонс вместо сути | начать с сути |
| «Раскрыть потенциал» | мотивационное клише | показать конкретный результат |
| «Вывести на новый уровень» | маркетинговый штамп | на какой уровень и чем измеряешь |
| «Открывает новые горизонты/перспективы» | штамп нулевой информативности | назвать конкретный горизонт или убрать |
| «Комплексный подход/решение» | канцелярит + раздувание | перечислить, что входит |
| «В связи с этим…» | формульная связка | убрать или показать причину-следствие |
| «От X до Y» (несвязанные понятия) | ложный диапазон | перечислить конкретно или убрать |
| «По своей сути», «В конечном счёте, главное…» | авторитетный трюизм без содержания | если верно без преамбулы — убрать преамбулу |

> Маркеры эволюционируют: слова, на которые обратили внимание, исчезают из AI-текстов («delve» упал после 2024). Список актуален на 2026 год, обновлять раз в квартал по Wikipedia «Признаки сгенерированности текста».

## Вычистка чат-бот-обвязки

Удалять всегда (кроме реальной переписки с клиентом): «Конечно!», «Отличный вопрос!», «Рад помочь!», «Надеюсь, это помогло», «Если остались вопросы — спрашивайте», «Дайте знать, если нужно развернуть», «Готов ответить на дополнительные вопросы». Автор статьи не «рад помочь».
