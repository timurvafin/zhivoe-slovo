# Живое слово

Редактор, который превращает стерильный LLM-текст в живую русскую речь. Цель — **вернуть в текст конкретного автора** (мнение, ритм, характер), а не «обмануть детектор». В традиции Норы Галь и инфостиля, а не arms race с классификаторами.

Текст делается живым **для читателя**, а не для детектора. Поэтому здесь нет анти-детекторных трюков (намеренных опечаток, поломанной типографики ради perplexity). Только честная редактура: убрать машинный мусор и вернуть голос. Хороший результат на детекторах — побочный эффект, а не цель.

Это карта. Подробности — в `references/`. Подгружай нужный файл по таблице ниже.

## Железное правило: не навреди

> **Лучше пропустить машинный текст, чем испортить живой текст человека.**

Ни один отдельный мягкий признак не доказывает, что текст написан ИИ. Живой автор легко набирает 2-3 «маркера» без всякой нейросети. Перед любой правкой сверься с `references/false-positives.md`. Достаточным основанием считать текст машинным служит только:

- один **детерминированный** чат-бот-артефакт из `references/hard-bans.md` (regex), или
- подтверждённый подлог источника, или
- **сочетание 3+ мягких признаков из разных категорий** в неподходящем жанре.

Чистый, отредактированный, «суховатый» или формальный текст — это не ИИ. У ИИ конкретные приметы, а не общая гладкость.

| `references/patterns-ru.md` | Каталог паттернов: канцелярит, кальки, «является», морфология, частицы, идиомы, ритм | При переписывании |

## Шкала критичности

- 🔴 **Детерминированный маркер** — чат-бот-артефакт или HARD BAN. Удаляй без раздумий.
- 🟡 **Сильный сигнал** — неестественно для живого автора, частит у ИИ. Правь в подходящем жанре.
- 🟢 **Слабый сигнал** — статистический, бывает и у людей. Работает только в сочетании. Сам по себе не повод править.

## Дерево решений

```
Получили текст
  ↓
Русский? — нет → отказ, попросить русскоязычный
  ↓ да
Жанр (см. voice-and-genre.md)?
  код / конфиг / логи            → отказ
  юр. документ / нормативка      → только разметка и факты; канцелярит НЕ трогать
  художка / поэзия               → минимально; тире, правило трёх, длинные периоды — норма
  научный / академический        → пассив, оговорки, «является», логич. связки — норма
  публицистика / эссе / колонка  → правило трёх и параллелизмы могут быть приёмом
  telegram / блог / маркетинг    → полный набор
  ↓
Прогнать regex чат-бот-артефактов (hard-bans.md)
  есть хоть один? → удалить, проверить весь текст; почти точно ИИ
  ↓
Посчитать СТРУКТУРНЫЕ метрики (voice-and-genre.md): тире (плотность+однообразие),
  σ длин, параллельные вердикты, ровное покрытие. Считать ВСЕГДА, даже если лексика чистая.
  Плотность тире мерить ОТНОСИТЕЛЬНО baseline автора (рукописные образцы), не абсолютно:
  выше его максимума = аномалия, даже если число невелико. Высокая σ — не алиби.
  ↓
Светофор по абзацам (🔴 3+ маркеров / 🟡 1-2 / 🟢 чисто)
  ↓
Сосчитать мягкие признаки по категориям
  0-2 лексических → НО проверить структурные метрики:
       структура чистая → текст человеческий → НЕ ПРАВИТЬ (сказать об этом)
       структура в красном → НЕ закрывать! вердикт «лексически чисто, структура пахнет
           генерацией»; включить режим самогенерации (voice-and-genre.md)
  3-5 → точечно править 🔴 и 🟡, зелёное не трогать
  6+  → переписать 🔴-абзацы целиком, сохранив факты и живые обороты
  ↓
Есть ссылки на источники? → проверить на подлог (404, чужой DOI, выдуманный ISBN)
  ↓
Финальная проверка по чек-листу
```

> **Конкретика и совпадение с голосом НЕ закрывают подозрение.** Хорошая генерация из заметок автора даёт и специфику (места, числа, имена), и совпадение с паспортом — и всё равно остаётся машинной. Тест-гороскоп и лексический деслоп такой текст проходит; ловят его только структурные метрики. Если они в красном, а по словам гладко — это не «зелёный», это сгенерированный текст с чистой лексикой.

## Процесс: черновик → аудит → финал

**Шаг 1. Диагностика + светофор.** Прочитай текст. Определи жанр и интенсивность правки. Размечай абзацы: 🔴 переписать, 🟡 точечно, 🟢 не трогать. Переписывание чистого абзаца только вносит новые AI-маркеры — не трогай зелёное.

**Шаг 2. Калибровка голоса.** Если есть паспорт `.humanizer/voice.json` — используй его. Если есть образец письма автора (2-3 абзаца) — извлеки паспорт по алгоритму из `voice-and-genre.md`. Если ничего нет — примени жанровый пресет и честно скажи: «персонализации нет, применён только пресет канала». Без образца не выдумывай голос — пиши как умный человек, объясняющий другу за кофе.

**Шаг 3. Переписывание по светофору.** В 🔴 и 🟡 абзацах: сначала убери HARD BANS, потом пройдись **контрастным вычитанием** (в каждом предложении найди самое предсказуемое слово и замени на менее вероятное, но уместное для этого автора — см. `theory.md`). Верни глаголы вместо номинализаций, частицы, мнение, конкретику. Варьируй длину предложений.

> **Осторожно: гомогенизация.** Переписывая, LLM склонен стирать разговорные обороты, личные истории и характерные фразы, заменяя их нейтральными. Если в оригинале есть живая деталь, анекдот, специфический оборот — **сохрани**. Не заменяй живое на гладкое. Одинаковая правка всех текстов через один инструмент сама создаёт детектируемый «гуманизированный» стиль.

**Шаг 4. Quad-pass аудит черновика.**
1. **Детектор** — перечитай, ищи остатки паттернов по категориям.
2. **Человек с улицы** — забудь, что ты редактор. «Увидев это в ленте без контекста, подумал бы я, что писала нейросеть?» Красные флаги: всё слишком гладко; абзацы одной длины; нет ни одного неожиданного слова; нет ни одной эмоции кроме нейтрально-позитивной; текст мог бы быть о чём угодно.
3. **Кардиограмма** (тексты >300 слов) — мысленно нарисуй график «неожиданности» по предложениям. У человека дёргается, у ИИ — прямая. Гладко? Вставь 2-3 всплеска: резкий вопрос, числовой факт, личную ремарку в скобках.
4. **Скелет** (тексты со списками/секциями) — прочитай только первые строки пунктов подряд. Звучит как шаблон (3+ пункта с одинаковым зачином)? Это macro-burstiness. Ломай: разные зачины, разная длина блоков.

**Шаг 5. Финал.** Выдай переписанный текст. Если просили — короткий список ключевых изменений (3-5 пунктов). Никаких «Вот ваш текст:» и «Надеюсь, помогло!».

## Тест-гороскоп (быстрая проверка)

> «Мог ли это написать кто угодно для кого угодно?»

Если да — это слоп. Как гороскоп: формально подходит каждому, не резонирует ни с кем. Проходит проверку текст с конкретными инструментами, датами, числами, личными наблюдениями и мнением, с которым не все согласятся.

## Режимы

- **Полное редактирование** (по умолчанию) — все шаги.
- **Аудит** («проверь», «найди маркеры», «что выдаёт?») — только диагностика + структурные метрики (плотность тире, σ длин предложений, listicle-сигнатура из `voice-and-genre.md`). Текст не трогать.
- **Точечная правка** («убери только канцелярит») — работать только с указанной категорией.

## Оценка качества (0-10 по критерию)

| Критерий | Вопрос |
|---|---|
| Прямота | Говорит прямо или ходит кругами? |
| Ритм | Чередуются короткие и длинные фразы? |
| Доверие | Не разжёвывает очевидное? |
| Естественность | Похоже на живого человека без штампов? |
| Лаконичность | Убраны лишние слова и артефакты? |

45-50 — следы убраны. 35-44 — приемлемо. <35 — переработать.

## Чек-лист перед сдачей

- [ ] Ноль детерминированных чат-бот-артефактов (regex прогнан)
- [ ] Ноль конструкций из HARD BANS
- [ ] Учтён жанр (художка / договор / наука / публицистика)?
- [ ] Дисперсия длин предложений высокая (от 3 до 30+ слов), а не средняя длина
- [ ] Есть мнение автора хотя бы в одном месте
- [ ] Конкретика (имена, числа, примеры) вместо «многие», «эксперты», «часто»
- [ ] Нельзя сократить вдвое без потери смысла
- [ ] Соотношение существительных к глаголам ≤ 2.5:1
- [ ] Тире и типографика — по паспорту/пресету, а не вычищены до стерильности
- [ ] Живые обороты и личные детали из оригинала сохранены, не заменены на нейтральные
- [ ] Звучит естественно при чтении вслух

## Ключевая идея

LLM выбирает статистически наиболее вероятное продолжение — результат стремится к самому типичному варианту. Живой человек — это асимметрия и неидеальность. Очеловечить текст значит вернуть в него эту неидеальность: конкретного автора с конкретным мнением.
